# VK Echo / PromptMaster Bot

Long Poll, **vkbottle**, сообщество ВКонтакте.

## Скрипты

| Файл | Описание |
|------|----------|
| **`vk_prompt_bot.py`** | Основной бот PromptMaster: приветствие, меню веток, пайплайн из `vk_console_bot`, уточнения с кнопкой «Готово», таймаут 3 мин. |
| **`vk_echo_trial.py`** | Пробный эхо-бот с лимитом ответов (как в ветке `develop`), без цепочки OpenAI. |

## Зависимости

```powershell
cd vk_echo_bot
pip install -r requirements.txt
```

Нужны пакеты из `requirements.txt` (в т.ч. `vkbottle`, `openai`, `python-dotenv`) и доступ к каталогу **`vk_console_bot`** рядом с `vk_echo_bot` (импорт логики).

## Переменные `.env`

Загружаются **корневой** `.env` репозитория и локальный `vk_echo_bot/.env` (приоритет у `.env.local`, если есть).

| Переменная | Где нужна |
|------------|-----------|
| `VK_GROUP_TOKEN` или `vk_group_token` | Обязательно для Long Poll |
| `OPENAI_API_KEY`, `OPENAI_BASE_URL`, `OPENAI_MODEL` | Для `vk_prompt_bot.py` |

Шаблон: `.env_test`.

## Запуск

```powershell
cd C:\zero_code\PromptMaster\vk_echo_bot
python vk_prompt_bot.py
```

В логах появится polling; в диалоге с сообществом нажмите **«Старт»**.

## Поведение `vk_prompt_bot.py` (кратко)

- **Старт** — приветствие без строки «Команды: …» в конце, внизу текста — абзац про серые кнопки; inline-меню: «написать текст», короткие серые ярлыки сценариев (заглушки), красный **«Стоп»**.  
- **Серые кнопки** — ответ «сценарий в разработке»; пояснение «в разработке» дублируется префиксом в тексте сообщений с этим меню.  
- **«Написать текст»** — ожидание одного сообщения с задачей, затем пайплайн с **`force_branch_1`** (без классификатора). Свободный текст — полный классификатор.  
- **JSON улучшителя** — отдельное сообщение без меню веток; следующее — вопрос про уточнение и клавиатура **«Готово»**.  
- **Таймаут 180 с** — сброс сессии, полное приветствие как в консоли (`format_welcome`) и только кнопка **«Старт»**.  
- Состояние (`PeerState`) **только в RAM**; после рестарта процесса/контейнера сессии не восстанавливаются.

## Docker

Для релиза в контейнере понадобится том с конфигом/секретами и (при появлении) **volume для БД**. Персистентное хранение сессий и истории в репозитории пока **не реализовано**.
